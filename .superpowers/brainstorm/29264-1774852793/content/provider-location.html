Where should AI provider config live?

You need a place to set API keys, base URLs, and manage models. Where should this UI live in the app?

A

New tab inside /settings

Add an "AI" section to the existing Settings page alongside Theme, Editor, Shortcuts. Full-width config panel with per-provider expandable sections.

Pros

  • Consistent with existing settings UX
  • Plenty of space for model browsing
  • Already has sidebar nav pattern

Cons

  • Requires navigating away from the main view
B

Dedicated panel in the agent sidebar

A "Configure" button/tab in the agent panel header. Slides in a config view inline — stays close to where you use it. Model picker, API key, base URL all in one place without leaving the main screen.

Pros

  • In-context — configure while connected to a DB
  • No navigation needed

Cons

  • Tight space in the sidebar panel
  • Model browser may feel cramped
C

Both — quick-edit in agent panel + full config in /settings

Agent panel has a small inline key/model picker for quick changes. Settings → AI has the full config: model browser, base URL, token limits, etc.

Pros

  • Best of both worlds
  • Power users get full config, casual users get quick access

Cons

  • Two places to maintain
  • More work to build
