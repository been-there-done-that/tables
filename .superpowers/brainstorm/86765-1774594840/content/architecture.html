Tool Call Flow — the bridge
1
Claude decides to call run_query → MCP server handler fires
2
Harness emits tool.started SSE → stores Promise keyed by toolCallId
3
Frontend receives tool.started → calls invoke("execute_query") in Tauri
4
Rust executes query → result returned to frontend
5
Frontend POST /tool-result/:id → harness resolves Promise → MCP returns result to Claude
6
Claude sees the data, continues reasoning, calls more tools or writes its answer
Key insight: The harness never touches the database directly. It's purely a bridge — the frontend owns all Tauri IPC calls. This means zero new Rust code needed. All 14 tools are implemented in TypeScript (frontend) on top of existing commands.
